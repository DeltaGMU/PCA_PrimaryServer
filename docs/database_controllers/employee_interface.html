Employee Interface#

This module contains the functions that interface with the database server to handle the processing of employee account records. Any code related to the handling of employing account records that require creating, reading, updating, or deleting data from the database server must use this interface module.

async server.lib.database_controllers.employee_interface.check_employee_has_records(employee_id: str, session: Optional[sqlalchemy.orm.session.Session] = None) → bool#

This utility method verifies if an employee account record has any associated saved time sheets. This method is useful in determining if an account is eligible to be deleted, since employee accounts with associated time sheets cannot be deleted.

Parameters

  • employee_id (str, required) – The ID of the employee.

  • session (Session, optional) – The database session used to retrieve employee timesheet records for verification.

Returns

True, if the employee has at least one timesheet record associated with the account.

Return type

bool

async server.lib.database_controllers.employee_interface.create_employee(pyd_employee: server.lib.data_models.employee.PydanticEmployeeRegistration, session: Optional[sqlalchemy.orm.session.Session] = None) → Dict[str, any]#

This method creates a new employee account with all associated employee information such as employee contact information and employee role, and inserts the records into the database. When a new employee account is created, the employee password is automatically generated and sent to the employee’s email. The employee may reset their password from the employee login portal.

Parameters

  • pyd_employee (PydanticEmployeeRegistration, required) – The set of information required to register a new employee account, represented by the PydanticEmployeeRegistration pydantic class.

  • session (Session, optional) – The database session used to insert the employee information into the database.

Returns

A JSON-Compatible dictionary containing the newly created employee account information.

Return type

Dict[str, any]

Raises

HTTPException – If any of the provided parameters are invalid, or a database error occurred during employee creation.

async server.lib.database_controllers.employee_interface.get_all_employees(session: Optional[sqlalchemy.orm.session.Session] = None) → List[server.lib.data_models.employee.Employee]#

This method is used to retrieve all the employee account records stored in the database. This includes all employee accounts that may be disabled.

Parameters

session (Session, optional) – The database session used to retrieve all the employee account records.

Returns

A list of all the employee account records in the database.

Return type

List[Employee]

async server.lib.database_controllers.employee_interface.get_employee(username: str, session: Optional[sqlalchemy.orm.session.Session] = None) → server.lib.data_models.employee.Employee#

This method is used to retrieve a single employee account record from the database provided either an employee ID or employee primary email. For accounts with the same primary email as another, only the account that was registered to that primary email first will be returned. It is highly advised to retrieve employees using an employee ID since they are guaranteed to be unique.

Parameters

  • username (str, required) – The employee ID or the primary email of the employee account.

  • session (Session, optional) – The database session used to retrieve the employee record.

Returns

The employee account record associated with the employee ID or primary email.

Return type

Employee

Raises

HTTPException – If any of the provided parameters are invalid, or the employee ID or employee email does not correspond to any employee account record.

async server.lib.database_controllers.employee_interface.get_employee_role(user: server.lib.data_models.employee.Employee, session: Optional[sqlalchemy.orm.session.Session] = None) → server.lib.data_models.employee_role.EmployeeRole#

This method is used to retrieve an employee account record’s employee role. If you need to retrieve the employee role without a reference to the employee account record, then consider retrieving that record first before using this method.

Parameters

EmployeeRole

List[str]

async server.lib.database_controllers.employee_interface.remove_employees(employee_ids: server.lib.data_models.employee.PydanticEmployeesRemoval | str, session: Optional[sqlalchemy.orm.session.Session] = None) → List[server.lib.data_models.employee.Employee]#

This method accepts one or more employee IDs and deletes the corresponding employee accounts from the database. Please note that employee accounts that have associated timesheet records cannot be deleted unless all timesheet records are removed.

Parameters

  • employee_ids (PydanticEmployeesRemoval | str, required) – A single employee ID or a list of employee IDs to delete the corresponding employee account(s).

  • session (Session, optional) – The database session used to delete one or more employee account records.

Returns

The list of employee account records that have been deleted from the database.

Return type

List[Employee]

Raises

HTTPException – If any of the provided parameters are invalid, or the employee has associated timesheet records.

async server.lib.database_controllers.employee_interface.update_employee(employee_id, pyd_employee_update: server.lib.data_models.employee.PydanticEmployeeUpdate, session: Optional[sqlalchemy.orm.session.Session] = None) → server.lib.data_models.employee.Employee#

This method updates a single employee account record with updated employee information. Employee account passwords cannot be changed or updated using this method. Upon successful submission of the employee record, an email is sent to the employee notifying them that their account has been updated.

Parameters

  • employee_id (str, required) – The ID of the employee.

  • pyd_employee_update (PydanticEmployeeUpdate, required) – The updated employee information that should be used.

  • session (Session, optional) – The database session used to update the employee record.

Returns

The employee record that has been updated.

Return type

Employee

async server.lib.database_controllers.employee_interface.update_employee_password(employee_id: str, current_password: str, new_password: str, session: Optional[sqlalchemy.orm.session.Session] = None) → server.lib.data_models.employee.Employee#

This method updates an existing employee account’s password with a new password. The new employee password is provided as plain-text to this method which is then hashed and salted before being entered into the database. Upon a successful password change, the employee is sent an email notifying them that their account password has been updated.

Parameters

  • employee_id (str, required) – The ID of the employee.

  • current_password (str, required) – The current plain-text password to the employee account.

  • new_password (str, required) – The new plain-text password that the employee account should use.

  • session (Session, optional) – The database session used to update the employee account’s password.

Returns

The employee account record that has had a password change.

Return type

Employee

Raises

HTTPException – If any of the provided parameters are invalid.

async server.lib.database_controllers.employee_interface.update_employees(employee_updates: Dict[str, server.lib.data_models.employee.PydanticEmployeeUpdate], session: Optional[sqlalchemy.orm.session.Session] = None) → List[server.lib.data_models.employee.Employee]#

This method updates one or more employee account records with updated employee information provided in the form of a dictionary consisting of employee ID keys and employee update information values. Upon successful submission of an employee record, an email is sent to the employee notifying them that their account has been updated.

Parameters

  • employee_updates (Dict[str, PydanticEmployeeUpdate]) – A dictionary containing employee update information values paired with employee ID keys.

  • session (Session, optional) – The database session used to update one or more employee account records.

Returns

A list of the employee account records that have been updated in the database.

Return type

List[Employee]

Raises

HTTPException – If any of the provided parameters are invalid.
