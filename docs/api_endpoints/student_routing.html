API - Student Routing#

This module consists of FastAPI routing for Students. This handles all the REST API logic for creating, destroying, and updating student-related data.

class server.web_api.routing.v1.student_routing.StudentsRouter#

Bases: object

The API router responsible for defining endpoints relating to students. The defined endpoints allow HTTP requests to conduct create, read, update, and delete tasks on student records.

class Create#

Bases: object

async static create_new_student(pyd_student: server.lib.data_models.student.PydanticStudentRegistration, token: str = Depends(OAuth2PasswordBearer), session=Depends(get_db_session))#

An endpoint that creates a new student record and adds it to the students’ table in the database.

Parameters

  • pyd_student (PydanticStudentRegistration, required) – The new student’s first name, last name, carpool number, parent full name, parent primary email, student grade, and optional parameters like parent secondary email and enabling email notifications.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.session, optional) – The database session to use to create a new student record in the database.

Returns

A response model containing the student object that was created and inserted into the database.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the request body contains any invalid parameters, or the data provided is formatted incorrectly.

class Delete#

Bases: object

async static delete_student(student_id: str, token: str = Depends(OAuth2PasswordBearer), session=Depends(get_db_session))#

An endpoint to remove a student record from the student’s table in the database. Removal of a student using this endpoint will permanently delete the student record from the database and all records related to the student record in other tables through a cascading delete.

Parameters

  • student_id (str, required) – The ID of the student that needs to be deleted.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.session, optional) – The database session to use to delete the student record.

Returns

A response model containing the student object that was deleted from the database.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the provided request body contains an invalid student ID, or if the student does not exist in the database.

async static delete_students(student_ids: server.lib.data_models.student.PydanticStudentsRemoval, token: str = Depends(OAuth2PasswordBearer), session=Depends(get_db_session))#

An endpoint to remove multiple student records from the student’s table in the database. Removal of multiple students using this endpoint will permanently delete the student records from the database and all records related to the student records in other tables through a cascading delete. To remove multiple student records, the student IDs must be provided in a list.

Parameters

  • student_ids (PydanticStudentsRemoval, required) – A list of IDs of students that needs to be deleted.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.session, optional) – The database session to use to delete the student record.

Returns

A response model containing the student object that was deleted from the database.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the provided request body contains an invalid student ID, or if the student does not exist in the database.
