API - Employee Routing#

This module consists of FastAPI routing for Employees. This handles all the REST API logic for creating, reading, updating, and destroying employee-related data.

async static register_new_employee(pyd_employee: server.lib.data_models.employee.PydanticEmployeeRegistration, token: str = Depends(OAuth2PasswordBearer), session=Depends(get_db_session))#

An endpoint to create a new employee entity and adds it to the employees’ table in the database.

Parameters

  • pyd_employee (PydanticEmployeeRegistration, required) – The Pydantic Employee Registration reference. This means that HTTP requests to this endpoint must include the required fields in the PydanticEmployeeRegistration.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.Session, optional) – The database session to use to register a new employee.

Returns

A response model containing the employee object that was created from the provided information with a generated employee ID and hashed password.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the provided request body contains any invalid parameters for the employee. This error may also be caused if the employee already exists in the database.

An endpoint to remove ALL employee records from the employee’s table in the database. Removal of multiple employees using this endpoint will permanently delete the employee records from the database and all records related to the employee records in other tables through a cascading delete.

Parameters

  • are_you_sure (str, required) – An additional parameter that must say ‘yes’ to fulfill this request, to prevent accidental deletion.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.session, optional) – The database session to use to delete the employee records.

Returns

A response model containing the number of employees deleted.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the provided request body contains invalid parameters.

async static delete_employees(employee_ids: server.lib.data_models.employee.PydanticEmployeesRemoval, token: str = Depends(OAuth2PasswordBearer), session=Depends(get_db_session))#

An endpoint to remove multiple employee records from the employee’s table in the database. Removal of multiple employees using this endpoint will permanently delete the employee records from the database and all records related to the employee records in other tables through a cascading delete. To remove multiple employee records, the employee IDs must be provided in a list.

Parameters

  • employee_ids (PydanticEmployeeRemoval, required) – A list of IDs of employees that needs to be deleted.

  • token (str, required) – The JSON Web Token responsible for authenticating the user to this endpoint.

  • session (sqlalchemy.orm.session, optional) – The database session to use to delete the employee record.

Returns

A response model containing the employee object that was deleted from the database.

Return type

server.web_api.models.ResponseModel

Raises

HTTPException – If the provided request body contains an invalid employee ID, or if the employee does not exist in the database.
