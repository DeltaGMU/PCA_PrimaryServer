Web Security Module#

This module contains utility methods to handle web security functionality for the API server. This includes generating access tokens, blacklisting tokens, etc.

async server.web_api.web_security.add_token_to_blacklist(token: str) → bool#

This utility method adds the provided access token to the access token blacklist. This is to prevent JWT hijacking so access tokens are blacklisted when they expire or the user account signs out.

Parameters

token (str, required) – The access token of the employee account that needs to be blacklisted.

Returns

True if the access token was successfully blacklisted.

Return type

bool

Raises

HTTPException – If the access token is invalid or expired, or the token has already been invalidated and blacklisted.

async server.web_api.web_security.create_access_token(employee_user: server.lib.data_models.employee.Employee) → Dict[str, str]#

A utility method used to generate an access token for an employee account upon successful sign-in. This method generates a JSON Web Token (JWT) with the current time as the issue time, expiration time as specified in the server configuration file, and includes the appropriate security scopes for the signed-in employee account.

Parameters

employee_user (Employee, required) – The employee record associated to the account that signed-in to the server.

Returns

A JSON-Compatible dictionary containing basic employee account information and access token information.

Return type

Dict[str, str]

async server.web_api.web_security.get_user_from_token(token: str, session=None) → server.lib.data_models.employee.Employee | None#

This utility method decodes a JSON Web Token (JWT) to retrieve the employee account ID. Using this ID, the employee record is retrieved from the database and returned.

Parameters

  • token (str, required) – The access token of the employee account.

  • session (Session, optional) – The database session used to retrieve the employee record.

Returns

None if there is a JWT decode error, otherwise the employee record associated with the employee ID decoded from the JWT.

Return type

Employee | None

async server.web_api.web_security.token_is_valid(token: str, scopes: List[str]) → bool#

This utility method verifies the validity of a user access token and ensures that the permissions scopes match those of the employee account. This utility method is used frequently for protected API routes to ensure only authenticated users can access the endpoints.

Parameters

  • token (str, required) – The access token of the employee account that needs to be verified.

  • scopes (List[str]) – A list of the permission scopes that the employee account must possess to be verified.

Returns

True if the access token corresponding to the employee account is valid and contains the correct permission scopes.

Return type

bool

Raises

HTTPException – If the access token is invalid or expired, or the employee account doesn’t have the appropriate permission scopes.
